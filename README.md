# Space Explorer - Short Reflection

## What API did you use?

I used the NASA APOD (Astronomy Picture of the Day) API, which is part of NASA's open APIs available at https://api.nasa.gov. This API provides access to NASA's daily astronomy images along with explanations from NASA scientists. The API requires an API key for access, with the DEMO_KEY providing limited requests for free. I utilized the APOD endpoint to fetch the astronomy picture of the day, and by modifying the count parameter, I was able to retrieve a collection of recent pictures for the gallery view. The API returns rich metadata including the image title, date, explanation text, copyright information, and both standard and high-resolution image URLs.

## What problem does your app solve?

Space Explorer solves the problem of making NASA's vast collection of astronomy imagery accessible to the general public in a mobile-friendly format. Space enthusiasts, students, and anyone curious about the cosmos often have to visit NASA's website directly to view these images, which may not provide the best experience on mobile devices. This app addresses that problem by presenting NASA's astronomy pictures in an intuitive gallery format that works seamlessly on smartphones. Users can browse through recent space images, search for specific topics within the picture titles and descriptions, filter to see only today's picture, and view detailed information about each image including the scientific explanation and copyright details. The app serves as a convenient gateway to explore the wonders of the universe directly from a mobile device, making space education more accessible to everyone.

## What was the most difficult part of the integration?

The most challenging part of integrating the NASA APOD API was handling the rate limiting that occurs when using the DEMO_KEY. NASA imposes strict hourly limits on API requests with the free demo key, which caused the app to fail with HTTP 503 errors when users exceeded these limits. Solving this required me to guide users to obtain their own free API key from NASA's website by filling out a simple registration form. Additionally, displaying the large astronomy images smoothly required implementing proper loading states withActivityIndicator components to show while images were loading. Another challenge was handling mixed media types, as the APOD API sometimes returns videos instead of images, which required filtering the results to only display image content in the gallery. Implementing pull-to-refresh functionality that works properly with React Navigation and maintaining good performance with large image lists also required careful consideration of state management and callback functions.

## What would you improve with more time?

With additional development time, I would implement several enhancements to make the Space Explorer app more comprehensive. First, I would add integration with additional NASA APIs such as the Mars Rover Photos API to allow users to view actual photos taken by rovers like Curiosity and Perseverance on the Red Planet. Second, I would implement local storage using AsyncStorage to cache previously viewed images, allowing users to build their own collection of favorites that can be accessed offline. Third, I would add date picker functionality so users can browse pictures from specific dates in the past rather than just the most recent ones. Fourth, I would enhance the filtering options to filter by year or by specific themes like nebulae, galaxies, or planets. Fifth, I would add video support for APOD entries that are videos rather than images, with a proper video player component. Sixth, I would implement image zooming and pinch-to-zoom functionality for the detail view to allow users to explore fine details in the high-resolution images. Seventh, I would add social sharing features optimized for different platforms and the ability to set pictures as wallpapers. Finally, I would improve the overall user interface with more sophisticated animations and transitions to create a more immersive space exploration experience.

## How to Run the App

To run Space Explorer, navigate to the api-quest folder in your terminal and run the command "npx expo start". Then scan the QR code with the Expo Go app on your phone, or press the "a" key to run on an Android emulator. Note that you will need a NASA API key from https://api.nasa.gov for the best experience, though the app will work with limited requests using the default DEMO_KEY. The app is built using React Native with Expo and uses React Navigation for screen navigation. All space images are fetched in real-time from NASA's APOD API.
